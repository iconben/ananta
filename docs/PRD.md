## 1. 产品概述

### 1.1 产品名称

| 语言 | 名称 |
|------|------|
| 中文 | 无量 |
| 藏文 | དཔག་མེད་ |
| 英文 | Ananta |
| 日文 | 無量（むりょう）|

名称取意：梵文 *ananta*（无终、无限），对应中文"无量"——一款专门计数的 App，却以"无量"为名，数而超越数，体现佛教的辩证思维。

### 1.2 产品定位

无量（Ananta）是一款面向个人佛教修行者的功课记录与共修社交 App。核心理念取自《华严经》"以微尘数劫精进不退"——每一次念诵、每一拜、每一盏灯，看似微小，积累无量。

### 1.3 目标用户

- 有日常修持习惯的佛教徒（汉传、藏传、南传均适用）
- 年龄跨度较大，包含较多中老年用户，需特别关注可读性和操作简便性
- 地域：中国大陆、海外华人、东南亚华人佛教圈

### 1.4 核心价值

- 记录并可视化个人修持积累
- 发起或参与共修活动，汇聚功德
- 与善友互相随喜、共同精进

## 2. 核心概念与数据模型

### 2.1 课目（Practice）

修持的具体内容，如观音心咒、大礼拜、药师经、供灯等。

- 永久存在，持续累积，无目标、无截止日期
- 每个课目有：名称、图标（emoji）、计量单位（遍/拜/部/盏/天等）、主题颜色
- 用户可自由新建、编辑、删除课目
- 删除课目时同步删除相关记录和发愿（需二次确认）

### 2.2 发愿（Campaign）

在某课目上立下的阶段性目标。

- 必须依附于某个课目
- 有明确的目标数量和截止日期
- 完成后状态变为"圆满"
- 来源：用户手动创建，或参加共修时自动生成
- 自动生成时名称格式：`共修名 · 课目名`

### 2.3 记录（Record）

每次修持的具体记录。

- 归属某课目（必选）
- 可选同时计入某发愿
- 同一条记录同时计入课目累计总数和发愿进度（不是二选一）
- 字段：日期时间、课目、数量、备注（可选）、所属发愿（可选）

### 2.4 共修活动（Retreat）

多人参与的阶段性共修。

- 任何用户均可发起
- 包含一个或多个课目，每个课目有建议目标数量（可选）
- 有开始/结束日期、加入权限（开放/需审核）、结束方式（自动/手动，默认自动）
- 参与者各自设定每个课目的目标数量（强制填写）
- 加入后自动为每个课目生成对应个人发愿
- 汇总数据匿名展示（只显示参与人数和合计数量，不展示个人数据）
- 通过 URL 或二维码分享

### 2.5 用户（User）

- 默认以匿名本地账号启动，无需注册即可使用
- 可选绑定邮箱 + 密码以实现云端备份和跨设备同步
- 忘记密码时通过邮箱 Magic Link 重置（链接15分钟有效）
- 完全不收集实名信息，不接入手机号/微信等实名认证
- 身份由用户自选的法名和头像字组成

### 2.6 好友关系（Friendship）

- 双向确认制（双方均同意才建立好友关系）
- 通过邀请码或共修活动链接相互认识
- 不支持通过真实姓名或手机号搜索

## 3. 功能模块

### 3.1 首页

#### 按发愿视图（默认）

- 进行中的发愿列表，每行显示：进度环、发愿名、所属课目标签、剩余天数
- 点击任意发愿进入**发愿详情页**
- 已圆满的发愿单独归为一组，可点击查看
- 底部全宽虚线按钮「＋ 新建发愿」

#### 按课目视图

- 课目卡片 3×N 网格，每格显示：图标、名称、累计总数、今日数量
- 点击课目卡片进入**课目详情页**
- 底部全宽虚线按钮「＋ 新建课目」

#### 课目详情页

- 标题：图标 + 课目名 + 累计总数
- 活跃热力图（近一年，颜色对应该课目主色）
- 相关发愿列表（含进度环和进度条）
  - 若有 2 个或以上发愿，底部显示跨发愿合并汇总
- 历史记录流水账（归属该课目的所有记录）
- 右上角编辑入口

#### 发愿详情页

- 标题：发愿名 + 所属课目标签 + 状态标签（剩X天/圆满）
- 大进度环（含百分比）+ 进度条 + 还差数量 + 日期区间
- 该发愿专属热力图（只统计计入此发愿的记录）
- 记录明细（只显示计入此发愿的条目）
- 右上角编辑入口

### 3.2 统计

- 时间维度切换：近12周 / 近6月 / 近3年
- 堆叠柱状图：各时间段各课目修持量，颜色区分课目
- 最近一期各课目汇总明细，含与上一期的涨跌幅（↑↓%）
- 点击首页课目卡片可进入该课目的历史记录明细

### 3.3 共修

#### 列表页

- 「我参与的共修」分组
- 「收到的共修邀请」分组（通过链接/二维码到达但尚未加入）
- 空状态时显示说明文字
- 底部全宽虚线按钮「＋ 发起共修」

#### 发起共修（Sheet）

- 名称（必填）、简介（可选）、起止日期（必填）
- 添加课目：从已有课目中选择，每个课目填建议目标数量（可选）
- 加入权限：开放加入 / 需审核（默认开放）
- 结束方式：到期自动结束 / 发起人手动结束（默认自动）
- 发起后自动为发起人生成对应发愿

#### 加入共修（通过链接/二维码）

- 展示活动详情（名称、课目、日期、参与人数、发起人）
- 对每个课目强制填写个人目标数量，可参考建议数量
- 确认后自动生成对应个人发愿
- 需审核模式：提交后显示等待审核状态

#### 共修详情页

- 活动信息：名称、发起人、日期、状态（进行中/已结束）、参与人数
- 活动简介
- 各课目匿名汇总进度（总人数 + 合计数量 + 建议目标参考线）
- 「我的发愿」卡片：显示每个课目个人进度，链接到对应发愿详情
- 未加入时显示「加入此共修」入口
- 右上角「分享」入口

#### 分享

- 专属 URL：`https://ananta.app/r/{id}`
- 二维码展示
- 注明加入权限（开放加入 / 需审核）
- 复制链接按钮

### 3.4 善友

#### 动态子 Tab

- 好友每日修持汇总卡片（按天聚合，不显示单条记录）
- 每张卡片显示：好友头像字、法名、日期、各课目数量标签
- 随喜按钮

#### 排行子 Tab

- 好友及自己的修持总量排行
- 每行显示：名次、头像字、法名、成就勋章、连续打卡天数、各课目数量标签
- 随喜按钮
- 自己的行高亮显示

#### 成就子 Tab

- 个人成就勋章展示（已解锁 + 待解锁）
- 连续打卡 Streak 可视化（21格火焰条）

#### 好友管理（右上角入口）

- 发送 / 接受 / 拒绝好友申请
- 通过邀请码或共修链接添加好友
- 好友列表
- 不支持实名搜索

### 3.5 设置（Header 右上角头像入口）

#### 个人信息

- 法名/显示名称
- 头像字（从预设汉字中选择）
- 个人简介（可选）

#### 字号大小

- 四档：小（×0.85）/ 中（×1.0，默认）/ 大（×1.2）/ 超大（×1.45）
- 全局缩放，直观预览，适配中老年用户需求

#### 隐私设置

- 修持数据是否公开（影响好友动态和排行榜可见性）
- 是否出现在排行榜
- 是否允许陌生人发送好友申请

#### 通知设置（占位，后期实现）

- 每日打卡提醒
- 共修活动通知
- 好友申请通知

#### 账号

- 绑定/更换邮箱
- 修改密码
- 数据导出（占位）
- 退出登录

## 4. 用户认证

### 4.1 流程

```
首次打开
  └→ 自动创建匿名本地账号（UUID）
  └→ 无需注册，直接进入 Onboarding

可选绑定（设置页提示）
  └→ 输入邮箱 + 设置密码
  └→ 本地数据上传云端，账号激活
  └→ 后续可跨设备登录

忘记密码
  └→ 输入邮箱
  └→ 收到 Magic Link 邮件（15分钟有效）
  └→ 点击链接 → 设置新密码
```

### 4.2 技术实现

- 自签发 JWT：access token（短期）+ refresh token（长期）
- Token 存储：Expo SecureStore
- 密码存储：bcrypt hash
- Magic Link：随机 token 存数据库，短期过期

### 4.3 隐私原则

- 完全不收集实名信息
- 不接入手机号、微信等实名认证方式
- 邮箱仅用于账号识别和找回，不做任何实名验证

---

## 5. 隐私设计原则

- 修持数据默认私密，用户主动选择分享
- Onboarding 时引导隐私设置，鼓励（但不强制）开放分享
- 好友关系双向确认，不支持通过真实身份搜索
- 共修汇总数据匿名展示，不暴露个人数据
- 后端不存储任何实名信息

## 6. 技术栈

### 6.1 前端
- **框架：** React Native + Expo（同一套代码支持 iOS、Android、Web）
- **路由：** Expo Router（文件系统路由）
- **状态管理：** Zustand
- **本地存储：** AsyncStorage（普通数据）+ SecureStore（Token）
- **Web 构建：** `npx expo export --web`

### 6.2 后端
- **运行时：** Node.js + TypeScript
- **框架：** Hono
- **ORM：** Drizzle ORM
- **数据库：** PostgreSQL

### 6.3 部署
- **服务器：** 阿里云香港节点
- **数据库：** 阿里云香港 RDS PostgreSQL
- **文件存储：** 阿里云 OSS（头像等静态资源）

### 6.4 Monorepo 结构
- **工具：** pnpm workspaces + Turborepo

